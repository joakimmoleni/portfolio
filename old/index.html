What I work on

I've been building things since I was five — custom PCs, LAN networks for hundreds, web dev before most people had email. After years running a business, I came back to what I always loved. Within my first week of COBOL training, I was deploying to production.

Today I write assembler at the deepest layer of one of Sweden's largest banks. COBOL got me in the door. Assembler is where I found my edge.

Production uptime requirement

  • Real money. Real-time.

    The code I write processes real financial flows — payments, settlements, core ledger operations. Not API calls to a payment provider — the actual assembler and COBOL paths where debits and credits are executed.

  • Downtime isn't a metric — it's a regulatory event

    The systems I work on can't go down. Not "shouldn't" — can't. Regulatory requirements, financial settlement windows, inter-bank obligations. Availability at this level isn't about load balancers. It's about writing code that is correct the first time.

  • Bridging 1970 and 2026

    I write assembler that was designed in the 1960s and JavaScript that shipped this week. The gap between those worlds is where I live — understanding both deeply enough to evolve one without destroying the other.

  • Clock cycles, not sprint cycles

    I optimise at the register level in assembler and build entire web platforms in vanilla JS. The problem doesn't care what layer it lives in, and neither do I. Performance isn't a metric I check after shipping — it's a constraint I design around.

What I work with daily

IBM z/OS, HLASM (assembler), COBOL, IMS, DB2, JCL, TSO/ISPF, Endevor. The mainframe stack — where uptime is contractual and "move fast and break things" gets you fired.

What I build with at night

Vanilla JavaScript, HTML, CSS. Custom routers, renderers, search engines, highlighters, build pipelines. No React. No Tailwind. No npm. Just understanding. I also own the IT and web for real businesses — from accounting firms to small companies that need things built properly.

What connects them

Systems thinking. The same engineer who optimises register usage in assembler also built a 15-file CSS architecture and an 18-module JavaScript application. The layer changes. The rigour doesn't.
